# Geo-distributed External Monitoring System

Реализация геораспределенной системы внешнего мониторинга для веб-приложения. Система осуществляет независимый сбор метрик доступности и производительности из трех географических точек и агрегирует данные в центральном хранилище.

## Архитектура и Компоненты системы

Архитектура построена по гибридной схеме Pull/Push: сбор метрик на местах происходит по Pull-модели, а передача в центр — по Push-модели через протокол `remote_write`.

### 1. Региональные узлы (Edge Nodes: MSK / NSK / VVO)
Точки мониторинга автономны — при обрыве связи с центром данные не теряются.

* **Prometheus (Edge)** — собирает метрики локально. Буферизирует данные при проблемах с сетью и потоково отправляет их в центр через `remote_write`.
* **Blackbox Exporter** — выполняет внешний мониторинг приложения (Blackbox-пробинг). Измеряет время DNS resolve, TCP connect, TLS handshake и HTTP transfer.
* **Node Exporter** — собирает базовые метрики самого сервера мониторинга (CPU, RAM, Disk, Network).

### 2. Центральный узел (Central Observability Core)
Сбор данных в одной точке, долгосрочное хранение и визуализация.

* **VictoriaMetrics (LTS)** — долгосрочное хранилище временных рядов. Принимает метрики от регионов и эффективно сжимает их для хранения.
* **Grafana** — отвечает за визуализацию. Обеспечивает быструю отрисовку дашбордов за счет прямого чтения из VictoriaMetrics.
* **Alertmanager** — фильтрует, группирует и маршрутизирует уведомления о сбоях.
* **PostgreSQL** — база данных для надежного хранения настроек и пользователей Grafana.


## Сетевая инфраструктура и Безопасность

* **WireGuard VPN** — организует защищенный полносвязный туннель между регионами и центром.
* Трафик `remote_write` полностью зашифрован. WireGuard работает на уровне ядра Linux, минимизируя сетевые задержки.


## Соответствие техническим требованиям (ФТ / НФТ)

* **Частота опроса (RPS):** Интервал скрейпинга — 60 секунд. Нагрузка составляет 0.016 RPS с точки (выполняет требование <= 0.1 RPS).
* **Глубина хранения (Retention):** Настроено хранение данных в течение 90 дней.
* **Скорость отрисовки:** Загрузка дашборда в Grafana занимает менее 5 секунд.
* **Метод мониторинга:** Только внешний пробинг (HTTP/HTTPS) без доступа к внутренним API.


## Оповещения (Alerting)

Настроены следующие сценарии уведомлений в Telegram:
* **WebsiteDown** — приложение недоступно (код не 2xx/3xx или таймаут) более 2 минут.
* **HighLatency** — время ответа приложения стабильно превышает SLA.
* **SSLExpiringSoon** — SSL-сертификат истекает менее чем через 14 дней.


## Развертывание и CI/CD

* **Docker Compose** — запуск всех сервисов в изолированных контейнерах.
* **Ansible** — раскатка конфигураций на серверы: установка Docker, настройка WireGuard, запуск контейнеров.
* **GitLab CI** — автоматическая проверка синтаксиса файлов при коммитах и деплой на серверы.
